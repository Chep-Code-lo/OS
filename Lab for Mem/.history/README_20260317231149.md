## 1. FIRST FIT - First_fit.cpp

#### 📌 Nguyên Tắc
- Tìm **block đầu tiên** từ đầu danh sách mà có kích thước ≥ kích thước file
- Gán file đó vào block đó
- Chuyển sang file tiếp theo

#### 📝 Giải Thích Code
```cpp
// Vòng lặp First Fit
for (m = 0; m < number_of_files; m++) {           // Duyệt từng file
    for (n = 0; n < number_of_blocks; n++) {      // Duyệt từng block
        if (block_arr[n] != 1) {                  // Block chưa được dùng?
            temp = blocks[n] - files[m];          // Tính fragment
            if (temp >= 0) {                      // Block đủ lớn?
                file_arr[m] = n;                  // Gán file vào block này
                break;                            // Dừng, tìm block đầu tiên rồi
            }
        }
    }
    fragments[m] = temp;                          // Lưu fragment
    block_arr[file_arr[m]] = 1;                   // Đánh dấu block đã dùng
}
```

**Ý tưởng triển khai:**
1. `block_arr[10]`: Mảng đánh dấu block đã dùng (0 = chưa dùng, 1 = đã dùng)
2. Duyệt từng file, với mỗi file duyệt từng block từ đầu
3. Nếu block chưa dùng và đủ lớn → gán file vào, tính fragment, đánh dấu đã dùng
4. **Dừng lại sau khi tìm được block đầu tiên** (break)

## 2. BEST FIT - Best_fit.cpp

#### 📌 Nguyên Tắc
- Tìm **block có phần dư nhỏ nhất**
- Duyệt hết tất cả các block có sẵn
- Chọn block mà `(block.size - file.size)` nhỏ nhất

#### 📝 Giải Thích Code
```cpp
for (int i = 0; i < np; i++) {          // Duyệt từng file
    int bestIdx = -1;                   // Block tốt nhất
    int minFragment = 1e9;              // Fragment nhỏ nhất

    for (int j = 0; j < nb; j++) {      // Duyệt tất cả block
        if (!used[j] && blocks[j] >= files[i]) {  // Block chưa dùng và đủ lớn
            int diff = blocks[j] - files[i];      // Tính fragment
            if (diff < minFragment) {             // Fragment nhỏ hơn?
                minFragment = diff;
                bestIdx = j;
            }
        }
    }

    if (bestIdx != -1) {
        allocation[i] = bestIdx;
        fragment[i] = minFragment;
        used[bestIdx] = true;
    }
}
```

**Ý tưởng triển khai:**
1. Duyệt từng file
2. Với mỗi file, duyệt **tất cả** các block để tìm block với fragment nhỏ nhất
3. Lưu `bestIdx` (block tốt nhất) khi fragment nhỏ hơn `minFragment`
4. Chỉ khi duyệt xong tất cả block mới gán file vào `bestIdx`

#### 📤 Output Ví Dụ
```
File    Size    Block    BlockSize    Fragment
1       50      1        100          50
2       180     3        200          20
3       95      2        150          55
```

**Giải thích output:**
- File 1 (50KB): So sánh fragment tất cả blocks (100-50=50, 150-50=100, 200-50=150) → Min = 50 → Block 1
- File 2 (180KB): Blocks còn (150, 200, 120) → So sánh (block 2 không đủ, 200-180=20, 120 không đủ) → Min = 20 → Block 3
- File 3 (95KB): Blocks còn (150, 120) → So sánh (150-95=55, 120-95=25) → Min = 25...

## 3. WORST FIT - Worst_fit.cpp

#### 📌 Nguyên Tắc
- Tìm **block có phần dư lớn nhất**
- Duyệt hết tất cả block có sẵn
- Chọn block mà `(block.size - file.size)` **lớn nhất**

#### 📝 Giải Thích Code (Điểm khác)
```cpp
// Khác biệt chính: Tìm fragment LỚN NHẤT thay vì nhỏ nhất

int top = 0;  // Lưu fragment lớn nhất hiện tại

for (m = 0; m < number_of_files; m++) {
    for (n = 0; n < number_of_blocks; n++) {
        if (block_arr[n] != 1) {
            temp = blocks[n] - files[m];
            if (temp >= 0) {
                if (top < temp) {          // ← So sánh: fragment > top?
                    file_arr[m] = n;
                    top = temp;            // Cập nhật fragment lớn nhất
                }
            }
        }
    }
}
```

**Ý tưởng triển khai:**
- Giống Best Fit nhưng tìm fragment **lớn nhất** thay vì nhỏ nhất
- Lưu block có phần dư lớn nhất để giữ block lớn cho file lớn sau

## 4. FIFO - FIFO.cpp

#### 📌 Nguyên Tắc
- Trang nào vào first → sẽ bị loại bỏ first
- Sử dụng circular queue cho vị trí victim

#### 📝 Giải Thích Code
```cpp
int victim = 0;         // Index của frame sẽ bị thay thế

for (int pageIndex = 0; pageIndex < size; pageIndex++) {
    // Bước 1: Tìm trang trong frames
    bool isFound = false;
    for (int i = 0; i < FRAMES_NUMBER; i++)
        if (frames[i] == pages[pageIndex]) {
            isFound = true;
            break;
        }

    // Bước 2-3: Nếu không tìm thấy - thay thế tại vị trí victim
    if (!isFound) {
        frames[victim] = pages[pageIndex];           // Thay thế
        victim = (victim + 1) % FRAMES_NUMBER;       // Chuyển sang frame tiếp theo
        pageFaults++;
    }
}
```

**Ý tưởng triển khai:**
1. `victim = (victim + 1) % FRAMES_NUMBER` xoay vòng để luôn thay thế tại frame tiếp theo
2. Trang nào vào frame trước sẽ bị thay thế trước
3. Không cần theo dõi lịch sử, chỉ dùng vị trí xoay vòng

#### 📤 Output Ví Dụ
```
Sequence: 1, 2, 3, 4, 1

1               1       -1      -1
2               1       2       -1
3               1       2       3
4               4       2       3
1               4       1       3

Number Of Page Faults = 5
```

**Giải thích output:**
- Bước 1: 1 → Frames: [1, -1, -1], victim=0 → MISS
- Bước 2: 2 → Frames: [1, 2, -1], victim=1 → MISS
- Bước 3: 3 → Frames: [1, 2, 3], victim=2 → MISS
- Bước 4: 4 → Frames: [4, 2, 3], victim=0 (thay frame 0 trang 1) → MISS (4 faults)
- Bước 5: 1 → Frames: [4, 1, 3], victim=1 (thay frame 1 trang 2) → MISS (5 faults)

## 5. LRU - LRU.cpp

#### 📌 Nguyên Tắc
- **Loại bỏ trang được sử dụng lâu nhất**
- Theo dõi **lần truy cập cuối cùng** của mỗi trang
- Trang có index truy cập **nhỏ nhất** sẽ bị loại bỏ

#### 📝 Giải Thích Code
```cpp
if (!hasFreeFrame) {
    int lastUse[FRAMES_NUMBER];
    
    // Tìm lần truy cập cuối cùng của mỗi trang
    for (int i = 0; i < FRAMES_NUMBER; i++)
        for (int p = pageIndex; p >= 0; p--)  // Tìm từ hiện tại ngược lại
            if (pages[p] == frames[i]) {
                lastUse[i] = p;        // Lưu index lần truy cập gần nhất
                break;
            }

    // Tìm frame có lastUse nhỏ nhất (lâu nhất)
    int victim = 0;
    for (int i = 0; i < FRAMES_NUMBER; i++)
        if (lastUse[i] < lastUse[victim])
            victim = i;

    frames[victim] = pages[pageIndex];
}
```

**Ý tưởng triển khai:**
1. Duyệt ngược từ `pageIndex` đến 0 để tìm lần truy cập **gần nhất** của mỗi trang
2. Lưu index vị trí đó vào `lastUse[i]`
3. Chọn trang có index **nhỏ nhất** = truy cập **lâu nhất** trong quá khứ

#### 📤 Output Ví Dụ
```
Sequence: 1, 2, 3, 4, 1

1               1       -1      -1
2               1       2       -1
3               1       2       3
4               4       2       3
1               4       1       3

Number Of Page Faults = 5
```

**Giải thích output:**
- Bước 1-3: Fill 3 frames → 3 misses
- Bước 4: Trang 4 không có. Tìm backward:
  - lastUse[0]=0 (trang 1 dùng ở index 0) ← MIN
  - lastUse[1]=1 (trang 2 dùng ở index 1)
  - lastUse[2]=2 (trang 3 dùng ở index 2)
  - Loại trang 1 → Frames: [4, 2, 3] → MISS (4 faults)
- Bước 5: Trang 1 không có. Tìm backward:
  - lastUse[0]=3 (trang 4 dùng ở index 3)
  - lastUse[1]=1 (trang 2 dùng ở index 1) ← MIN
  - lastUse[2]=2 (trang 3 dùng ở index 2)
  - Loại trang 2 → Frames: [4, 1, 3] → MISS (5 faults)

## 6. LFU - LFU.cpp

#### 📌 Nguyên Tắc
- **Loại bỏ trang được sử dụng ít nhất**
- Theo dõi **số lần truy cập** của mỗi trang
- Trang có **số lần dùng ít nhất** sẽ bị loại bỏ

#### 📝 Giải Thích Code
```cpp
if (!hasFreeFrame) {
    int countUse[FRAMES_NUMBER] = {0};
    
    // Đếm số lần mỗi trang được truy cập
    for (int i = 0; i < FRAMES_NUMBER; i++)
        for (int p = pageIndex; p >= 0; p--)  // Duyệt tất cả trước đó
            if (pages[p] == frames[i])
                countUse[i]++;  // Tăng đếm

    // Tìm frame có countUse nhỏ nhất
    int victim = 0;
    for (int i = 0; i < FRAMES_NUMBER; i++)
        if (countUse[i] < countUse[victim])
            victim = i;

    frames[victim] = pages[pageIndex];
}
```

**Ý tưởng triển khai:**
1. Đếm **tất cả** lần xuất hiện của trang trong quá khứ
2. Không dừng ở lần đầu tiên, tiếp tục tìm để đếm tất cả
3. Chọn trang có **số lần dùng ít nhất**

#### 📤 Output Ví Dụ
```
Sequence: 1, 2, 3, 4, 1

1               1       -1      -1
2               1       2       -1
3               1       2       3
4               4       2       3
1               1       2       3

Number Of Page Faults = 5
```

**Giải thích output:**
- Bước 1-3: Fill frames, countUse: [1, 1, 1] → 3 misses
- Bước 4: countUse bằng nhau tất cả. Chọn victim=0 (trang 1) → Frames: [4, 2, 3] → MISS (4 faults)
- Bước 5: Trang 1 không có. countUse: [1, 1, 1] bằng nhau → Chọn victim=0 (trang 4) → Frames: [1, 2, 3] → MISS (5 faults)

## 8. OPTIMAL - Optimal.cpp

#### 📌 Nguyên Tắc
- **Loại bỏ trang sẽ không được sử dụng lâu nhất** trong tương lai
- Tìm **lần truy cập tiếp theo** của mỗi trang
- Trang có **lần truy cập tiếp theo xa nhất** (hoặc không bao giờ dùng) sẽ bị loại bỏ

#### 📝 Giải Thích Code
```cpp
if (!hasFreeFrame) {
    int nextUse[FRAMES_NUMBER];
    for (int i = 0; i < FRAMES_NUMBER; i++)
        nextUse[i] = size + 1;  // Mặc định: không bao giờ sử dụng

    // Tìm lần truy cập tiếp theo của mỗi trang
    for (int i = 0; i < FRAMES_NUMBER; i++)
        for (int p = pageIndex; p < size; p++)  // Tìm từ hiện tại trở đi
            if (pages[p] == frames[i]) {
                nextUse[i] = p;  // Lưu index lần truy cập tiếp theo
                break;
            }

    // Tìm frame có nextUse lớn nhất (muộn nhất / không bao giờ)
    int victim = 0;
    for (int i = 0; i < FRAMES_NUMBER; i++)
        if (nextUse[i] > nextUse[victim])
            victim = i;

    frames[victim] = pages[pageIndex];
}
```

**Ý tưởng triển khai:**
1. Duyệt **tương lai** để tìm lần truy cập tiếp theo
2. Lưu index vị trí đó vào `nextUse[i]`
3. Nếu trang không xuất hiện lại, `nextUse[i]` = `size + 1`
4. Chọn trang có `nextUse` **lớn nhất** = sẽ được sử dụng **muộn nhất**

#### 📤 Output Ví Dụ
```
Sequence: 1, 2, 3, 4, 1

1               1       -1      -1
2               1       2       -1
3               1       2       3
4               1       4       3
1

Number Of Page Faults = 4
```

**Giải thích output:**
- Bước 1-3: Fill 3 frames → 3 misses
- Bước 4: Trang 4 không có. Tìm forward:
  - nextUse[0]=4 (trang 1 sẽ dùng ở index 4)
  - nextUse[1]=∞ (trang 2 sẽ không dùng)
  - nextUse[2]=∞ (trang 3 sẽ không dùng)
  - Max = ∞ → Chọn victim=1 (loại trang 2) → Frames: [1, 4, 3] → MISS (4 faults)
- Bước 5: Trang 1 đã có → **HIT!**

**So sánh:**
- Optimal biết tương lai → Loại trang không dùng, giữ trang sẽ dùng
- **4 page faults = THẤP NHẤT** (FIFO/LRU/LFU = 5)
- **Không dùng thực tế**: Không thể dự đoán tương lai
