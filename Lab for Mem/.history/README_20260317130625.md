# Giải Thích Chi Tiết Code C++ và Output

## 1. FIRST FIT - First_fit.cpp

#### 📌 Nguyên Tắc
Các thuật toán này được sử dụng để phân bổ các block bộ nhớ cho các file/process cần cấp phát. Chúng giải quyết vấn đề: **Khi có một file với kích thước X cần được lưu trữ, ta nên sử dụng block nào?**

**Các khái niệm quan trọng:**
- **Block**: Các vùng bộ nhớ có sẵn
- **File**: Các file cần được lưu trữ
- **Fragment (phân mảnh)**: Phần bộ nhớ không sử dụng trong block sau khi lưu file
  - Fragment = Kích thước block - Kích thước file

---

### 1. FIRST FIT (Thích Hợp Đầu Tiên)

#### 📌 Nguyên Tắc
- Tìm **block đầu tiên** từ đầu danh sách mà có kích thước ≥ kích thước file
- Gán file đó vào block đó
- Chuyển sang file tiếp theo

#### ✅ Ưu Điểm
- **Nhanh nhất**: Không cần tìm kiếm toàn bộ danh sách
- **Hiệu quả**: Phù hợp cho các hệ thống có yêu cầu tốc độ cao
- **Đơn giản**: Dễ triển khai

#### ❌ Nhược Điểm
- **Phân mảnh ngoài cao**: Để lại nhiều fragment không sử dụng được
- **Sử dụng bộ nhớ kém hiệu quả**: Có thể lãng phí bộ nhớ

#### 💻 Ví Dụ
```
Blocks:    [20, 15, 30, 35]
Files:     [10, 20, 30]

File 1 (10): Thích hợp đầu tiên → Block 1 (20) ✓ Fragment = 10
File 2 (20): Thích hợp đầu tiên → Block 2 (15) ✗ Quá nhỏ
             Thích hợp đầu tiên → Block 3 (30) ✓ Fragment = 10
File 3 (30): Thích hợp đầu tiên → Block 4 (35) ✓ Fragment = 5

Kết quả: Tổng fragment = 25
```

#### 📝 Mã Giả
```
for each file f in files:
    for each block b in blocks:
        if b not used AND b.size >= f.size:
            allocate f to b
            mark b as used
            break
```

---

### 2. BEST FIT (Thích Hợp Tốt Nhất)

#### 📌 Nguyên Tắc
- Tìm **block có phần dư nhỏ nhất** (gần nhất với kích thước file)
- Duyệt hết tất cả các block có sẵn
- Chọn block mà `(block.size - file.size)` nhỏ nhất
- Mục đích: Giảm thiểu phân mảnh ngoài

#### ✅ Ưu Điểm
- **Phân mảnh nhỏ**: Giảm fragment không sử dụng được
- **Bộ nhớ hiệu quả**: Sử dụng bộ nhớ tốn ít lãng phí hơn
- **Tối ưu**: Thường cho kết quả tốt hơn First Fit

#### ❌ Nhược Điểm
- **Chậm hơn**: Phải duyệt tất cả block để so sánh
- **Bộ nhớ chứa đầy**: Nhiều block nhỏ không sử dụng được
- **Overhead cao**: Tốn thời gian xử lý để tìm block tốt nhất

#### 💻 Ví Dụ
```
Blocks:    [20, 15, 30, 35]
Files:     [10, 20, 30]

File 1 (10): 
  - Block 1: 20 - 10 = 10 (fragment)
  - Block 2: 15 - 10 = 5  (fragment) ← Tốt nhất
  - Block 3: 30 - 10 = 20 (fragment)
  - Block 4: 35 - 10 = 25 (fragment)
  Chọn Block 2 → Fragment = 5

File 2 (20):
  - Block 1: 20 - 20 = 0 (fragment) ← Tốt nhất
  - Block 3: 30 - 20 = 10 (fragment)
  - Block 4: 35 - 20 = 15 (fragment)
  Chọn Block 1 → Fragment = 0

File 3 (30):
  - Block 3: 30 - 30 = 0 (fragment) ← Tốt nhất
  - Block 4: 35 - 30 = 5 (fragment)
  Chọn Block 3 → Fragment = 0

Kết quả: Tổng fragment = 5 (ít hơn First Fit!)
```

#### 📝 Mã Giả
```
for each file f in files:
    bestBlock = -1
    minFragment = infinity
    
    for each block b in blocks:
        if b not used AND b.size >= f.size:
            fragment = b.size - f.size
            if fragment < minFragment:
                minFragment = fragment
                bestBlock = b
    
    if bestBlock != -1:
        allocate f to bestBlock
        mark bestBlock as used
```

---

### 3. WORST FIT (Thích Hợp Tồi Nhất)

#### 📌 Nguyên Tắc
- Tìm **block có phần dư lớn nhất** (hiện tại)
- Duyệt hết tất cả block có sẵn
- Chọn block mà `(block.size - file.size)` **lớn nhất**
- Mục đích: Để lại các block lớn cho các file lớn trong tương lai

#### ✅ Ưu Điểm
- **Giữ block lớn**: Còn lại block lớn cho file lớn sau này
- **Hiệu quả trung hạn**: Có thể hữu ích nếu file đến không dự đoán được
- **Phân mảnh ngoài hợp lý**: Cân bằng giữa fragment

#### ❌ Nhược Điểm
- **Chậm**: Phải duyệt tất cả block
- **Fragment lớn**: Để lại nhiều phần dư không sử dụng
- **Thực tiễn không tốt**: Ít khi tốt hơn Best Fit

#### 💻 Ví Dụ
```
Blocks:    [20, 15, 30, 35]
Files:     [10, 20, 30]

File 1 (10):
  - Block 1: 20 - 10 = 10 (fragment)
  - Block 2: 15 - 10 = 5  (fragment)
  - Block 3: 30 - 10 = 20 (fragment)
  - Block 4: 35 - 10 = 25 (fragment) ← Lớn nhất
  Chọn Block 4 → Fragment = 25

File 2 (20):
  - Block 1: 20 - 20 = 0 (fragment) ← Lớn nhất trong những block còn lại
  - Block 2: 15 - 20 = ✗ (quá nhỏ)
  - Block 3: 30 - 20 = 10 (fragment)
  Chọn Block 3 → Fragment = 10

File 3 (30):
  - Block 1: 20 - 30 = ✗ (quá nhỏ)
  - Block 2: 15 - 30 = ✗ (quá nhỏ)
  Không thể cấp phát! ✗

Kết quả: Tổng fragment = 35 (tệ nhất!)
```

#### 📝 Mã Giả
```
for each file f in files:
    worstBlock = -1
    maxFragment = -1
    
    for each block b in blocks:
        if b not used AND b.size >= f.size:
            fragment = b.size - f.size
            if fragment > maxFragment:
                maxFragment = fragment
                worstBlock = b
    
    if worstBlock != -1:
        allocate f to worstBlock
        mark worstBlock as used
```

### Bảng So Sánh Thuật Toán Cấp Phát Bộ Nhớ

| Tiêu Chí | First Fit | Best Fit | Worst Fit |
|----------|-----------|----------|-----------|
| **Tốc độ** | ⭐⭐⭐⭐⭐ Nhanh nhất | ⭐⭐⭐ Chậm | ⭐⭐⭐ Chậm |
| **Fragment** | ⭐⭐ Cao | ⭐⭐⭐⭐ Thấp | ⭐ Rất cao |
| **Sử dụng bộ nhớ** | ⭐⭐ Kém | ⭐⭐⭐⭐⭐ Tốt | ⭐ Rất kém |
| **Khó áp dụng** | ⭐ Dễ | ⭐⭐ Vừa | ⭐⭐ Vừa |
| **Ứng dụng thực tế** | ⭐⭐⭐⭐ Tốt | ⭐⭐⭐⭐⭐ Tốt nhất | ⭐ Hiếm dùng |

## II. THUẬT TOÁN THAY THẾ TRANG (Page Replacement)

### Khái Niệm Chung
Các thuật toán này được sử dụng trong **Virtual Memory** (Bộ nhớ ảo) để quyết định **trang nào sẽ bị loại bỏ** khi bộ nhớ đầy và cần đưa trang mới vào.

**Các khái niệm quan trọng:**
- **Trang (Page)**: Đơn vị quản lý bộ nhớ
- **Khung (Frame)**: Vị trí trong bộ nhớ vật lý
- **Page Fault**: Tình trạng trang cần thiết không có trong memory
- **Victim**: Trang sẽ bị loại bỏ khi memory đầy

#### Hàm Kiểm Tra Page Fault
```cpp
bool isInFrames(int page, int frames[], int FRAMES_NUMBER) {
    for (int i = 0; i < FRAMES_NUMBER; i++) {
        if (frames[i] == page) return true;
    }
    return false;
}
```

---

### 1. FIFO (First In First Out)

#### 📌 Nguyên Tắc
- Giữ thứ tự: Trang nào được đưa vào frame trước thì sẽ bị loại bỏ trước
- Sử dụng cái gọi là **"từng cặp tay"** (circular queue)
- Trang đầu tiên được đưa vào sẽ là trang đầu tiên bị loại bỏ
- Mục đích: Công bằng cho tất cả các trang

#### ✅ Ưu Điểm
- **Đơn giản**: Dễ hiểu và dễ triển khai
- **Rẻ**: Chi phí xử lý thấp
- **Công bằng**: Tất cả trang được đối xử như nhau

#### ❌ Nhược Điểm
- **Belady's Anomaly**: Tăng số frame nhưng lại tăng page fault
- **Không thông minh**: Không xét tính sử dụng của trang
- **Hiệu suất kém**: Có thể loại bỏ trang được sử dụng thường xuyên

#### 💻 Ví Dụ & Hoạt Động Từng Bước
```
Trang yêu cầu: [1, 2, 3, 1, 4, 5, 2, 1, 2, 6, 7, 3, 2]
Số frame = 3
Victim pointer = 0 (bắt đầu từ frame 0)

Bước 1: Trang 1 → Thêm vào frame 0 → [1, _, _] → Page Fault
Bước 2: Trang 2 → Thêm vào frame 1 → [1, 2, _] → Page Fault
Bước 3: Trang 3 → Thêm vào frame 2 → [1, 2, 3] → Page Fault
Bước 4: Trang 1 → Đã có trong frame → Không làm gì
Bước 5: Trang 4 → Thay thế frame 0 (trang 1) → [4, 2, 3] → Page Fault
Bước 6: Trang 5 → Thay thế frame 1 (trang 2) → [4, 5, 3] → Page Fault
Bước 7: Trang 2 → Thay thế frame 2 (trang 3) → [4, 5, 2] → Page Fault
Bước 8: Trang 1 → Thay thế frame 0 (trang 4) → [1, 5, 2] → Page Fault
Bước 9: Trang 2 → Đã có trong frame → Không làm gì
Bước 10: Trang 6 → Thay thế frame 1 (trang 5) → [1, 6, 2] → Page Fault
Bước 11: Trang 7 → Thay thế frame 2 (trang 2) → [1, 6, 7] → Page Fault
Bước 12: Trang 3 → Thay thế frame 0 (trang 1) → [3, 6, 7] → Page Fault
Bước 13: Trang 2 → Thay thế frame 1 (trang 6) → [3, 2, 7] → Page Fault

Tổng Page Faults: 11
```

#### 📝 Mã Giả
```
victim = 0  // index của trang sẽ bị loại bỏ

for each page p in pages:
    if page p is in frames:
        // Không làm gì
    else:
        if has free frame:
            // Thêm vào frame rỗi
        else:
            // Loại bỏ trang ở vị trí victim
            frames[victim] = p
            victim = (victim + 1) % FRAMES_NUMBER
        page_faults++
```

---

### 2. LRU (Least Recently Used)

#### 📌 Nguyên Tắc
- **Loại bỏ trang được sử dụng lâu nhất**
- Theo dõi **lần truy cập cuối cùng** của mỗi trang
- Trang mà lần truy cập cuối cùng **gần nhất** (chỉ số nhỏ nhất) sẽ bị loại bỏ
- Mục đích: Giữ trang được sử dụng gần đây

#### ✅ Ưu Điểm
- **Thông minh**: Xét thời gian sử dụng của trang
- **Hiệu suất tốt**: Thường cho kết quả tốt trong thực tế
- **Hợp lý**: Dựa trên giả thiết "locality of reference"
- **Không Belady**: Không bị Belady's Anomaly

#### ❌ Nhược Điểm
- **Chi phí cao**: Phải theo dõi thời gian truy cập của mỗi trang
- **Phức tạp**: Khó triển khai hiệu quả
- **Overhead**: Tốn thời gian và bộ nhớ để lưu thông tin

#### 💻 Ví Dụ & Hoạt Động Từng Bước
```
Trang yêu cầu: [1, 2, 3, 1, 4, 5, 2, 1, 2, 6, 7, 3, 2]
Số frame = 3

Bước 1: Trang 1 → [1, _, _] → Page Fault
Bước 2: Trang 2 → [1, 2, _] → Page Fault
Bước 3: Trang 3 → [1, 2, 3] → Page Fault
Bước 4: Trang 1 → Đã có, cập nhật lastUse[0] = 3
Bước 5: Trang 4 → Loại bỏ trang 2 (lastUse = 1, nhỏ nhất) → [1, 4, 3] → Page Fault
Bước 6: Trang 5 → Loại bỏ trang 3 (lastUse = 2, nhỏ nhất) → [1, 4, 5] → Page Fault
Bước 7: Trang 2 → Loại bỏ trang 4 (lastUse = 4, nhỏ nhất) → [1, 2, 5] → Page Fault
Bước 8: Trang 1 → Đã có, cập nhật lastUse = 7
Bước 9: Trang 2 → Đã có, cập nhật lastUse = 8
Bước 10: Trang 6 → Loại bỏ trang 5 (lastUse = 5, nhỏ nhất) → [1, 2, 6] → Page Fault
Bước 11: Trang 7 → Loại bỏ trang 1 (lastUse = 7, nhỏ nhất) → [7, 2, 6] → Page Fault
Bước 12: Trang 3 → Loại bỏ trang 6 (lastUse = 9, nhỏ nhất) → [7, 2, 3] → Page Fault
Bước 13: Trang 2 → Đã có

Tổng Page Faults: 9 (ít hơn FIFO)
```

#### 📝 Mã Giả
```
for each page p in pages:
    if page p is in frames:
        // Cập nhật thời gian truy cập
    else:
        if has free frame:
            // Thêm vào frame rỗi
        else:
            // Tìm trang có lastUse nhỏ nhất
            victim = frame với lastUse nhỏ nhất
            frames[victim] = p
        page_faults++
```

### 3. LFU (Least Frequently Used)

#### 📌 Nguyên Tắc
- **Loại bỏ trang được sử dụng ít nhất**
- Theo dõi **số lần truy cập** của mỗi trang
- Trang có **số lần truy cập ít nhất** sẽ bị loại bỏ
- Mục đích: Giữ lại trang được sử dụng thường xuyên

#### ✅ Ưu Điểm
- **Dựa trên tần suất**: Xét số lần sử dụng
- **Tối ưu**: Loại bỏ trang ít hữu ích
- **Hiệu quả**: Tốt cho các trang có hành vi sử dụng không đều

#### ❌ Nhược Điểm
- **Chi phí cao**: Phải đếm số lần truy cập
- **Không xét thời gian**: Trang cũ được sử dụng nhiều không bị loại bỏ
- **Phức tạp**: Khó triển khai hiệu quả

#### 💻 Ví Dụ & Hoạt Động Từng Bước
```
Trang yêu cầu: [1, 2, 3, 1, 4, 5, 2, 1, 2, 6, 7, 3, 2]
Số frame = 3

Bước 1: Trang 1 → [1, _, _] → Count: [1] → Page Fault
Bước 2: Trang 2 → [1, 2, _] → Count: [1, 1] → Page Fault
Bước 3: Trang 3 → [1, 2, 3] → Count: [1, 1, 1] → Page Fault
Bước 4: Trang 1 → Đã có → Count: [2, 1, 1]
Bước 5: Trang 4 → Loại trang 2 hoặc 3 (count=1) → [1, 4, 3] → Count: [2, 1, 1] → Page Fault
Bước 6: Trang 5 → Loại trang 3 (count=1) → [1, 4, 5] → Count: [2, 1, 1] → Page Fault
Bước 7: Trang 2 → Loại trang 4 hoặc 5 (count=1) → [1, 2, 5] → Count: [2, 1, 1] → Page Fault
Bước 8: Trang 1 → Đã có → Count: [3, 1, 1]
Bước 9: Trang 2 → Đã có → Count: [3, 2, 1]
Bước 10: Trang 6 → Loại trang 5 (count=1) → [1, 2, 6] → Count: [3, 2, 1] → Page Fault
Bước 11: Trang 7 → Loại trang 6 (count=1) → [1, 2, 7] → Count: [3, 2, 1] → Page Fault
Bước 12: Trang 3 → Loại trang 7 (count=1) → [1, 2, 3] → Count: [3, 2, 1] → Page Fault
Bước 13: Trang 2 → Đã có → Count: [3, 3, 1]

Tổng Page Faults: 9 (bằng LRU)
```

#### 📝 Mã Giả
```
for each page p in pages:
    if page p is in frames:
        countUse[frame_index]++
    else:
        if has free frame:
            // Thêm vào frame rỗi
        else:
            // Tìm trang có countUse nhỏ nhất
            victim = frame với countUse nhỏ nhất
            frames[victim] = p
        page_faults++
```

### 4. MFU (Most Frequently Used)

#### 📌 Nguyên Tắc
- **Loại bỏ trang được sử dụng nhiều nhất**
- Theo dõi **số lần truy cập** của mỗi trang
- Trang có **số lần truy cập nhiều nhất** sẽ bị loại bỏ
- Mục đích: Giả định rằng trang được sử dụng nhiều sắp sử dụng xong
- ⚠️ **Hiếm dùng**: Không thực tế, thường cho kết quả tệ

#### ✅ Ưu Điểm
- **Lý thuyết**: Dựa trên giả thiết hợp lý (mặc dù thường không đúng)
- **Đơn giản**: Dễ hiểu khái niệm

#### ❌ Nhược Điểm
- **Hiệu suất tệ**: Loại bỏ trang hữu ích nhất
- **Không thực tế**: Giả thiết thường sai trong thực tế
- **Kết quả tệ**: Thường cho kết quả tồi tệ nhất

#### 💻 Ví Dụ & Hoạt Động Từng Bước

### 5. OPTIMAL (Tối Ưu)

#### 📌 Nguyên Tắc
- **Loại bỏ trang sẽ không được sử dụng lâu nhất** (hoặc không bao giờ)
- Tìm **lần truy cập tiếp theo** của mỗi trang trong tương lai
- Trang có **lần truy cập tiếp theo xa nhất** sẽ bị loại bỏ
- Mục đích: **Tối ưu lý thuyết** (thường không thực hiện được trong thực tế)
- ⚠️ **Không thực tế**: Cần biết tương lai

#### ✅ Ưu Điểm
- **Tối ưu về mặt lý thuyết**: Cho số page fault ít nhất
- **Chuẩn so sánh**: Dùng để đánh giá các thuật toán khác
- **Hiệu suất tốt nhất**: Trên lý thuyết

#### ❌ Nhược Điểm
- **Không thực tế**: Cần biết tương lai (không thể biết trước)
- **Không triển khai được**: Không thể sử dụng trong thực tế
- **Chỉ để tham khảo**: Dùng để so sánh với các thuật toán khác

#### 💻 Ví Dụ & Hoạt Động Từng Bước
```
Trang yêu cầu: [1, 2, 3, 1, 4, 5, 2, 1, 2, 6, 7, 3, 2]
Số frame = 3

Lần truy cập tiếp theo:
- Bước 5: Trang 1 → Tiếp theo ở bước 7 → Index = 7
- Bước 5: Trang 2 → Tiếp theo ở bước 6 → Index = 6
- Bước 5: Trang 3 → Tiếp theo ở bước 12 → Index = 12
→ Loại bỏ trang 2 (gần nhất)

Bước 1: Trang 1 → [1, _, _] → Page Fault
Bước 2: Trang 2 → [1, 2, _] → Page Fault
Bước 3: Trang 3 → [1, 2, 3] → Page Fault
Bước 4: Trang 1 → Đã có

Bước 6: Trang 5 → Loại 1 (tiếp theo=7) → [5, 4, 3] → Page Fault
Bước 7: Trang 2 → Loại 4 (tiếp theo=∞) → [5, 2, 3] → Page Fault

```

#### 📝 Mã Giả
```
for each page p in pages[current]:
    if page p is in frames:
        // Không làm gì
    else:
        if has free frame:
            // Thêm vào frame rỗi
        else:
            // Tìm lần truy cập tiếp theo xa nhất
            for each frame in frames:
                find next occurrence of frame's page in pages[current+1:]
                store nextUse[frame_index]
            
            victim = frame với nextUse lớn nhất (xa nhất/không bao giờ)
            frames[victim] = p
        page_faults++
```

## III. ĐỊNH NGHĨA CÁC KHÁ NIỆM QUAN TRỌNG

### Page Fault
- **Khái niệm**: Khi CPU cần một trang nhưng trang đó không có trong memory (chỉ có trên disk)
- **Hậu quả**: Phải đọc từ disk, làm chậm hệ thống
- **Mục tiêu**: Giảm số page fault

### Fragment (Phân Mảnh)
- **Phân mảnh ngoài (External Fragmentation)**: Khoảng trống ngoài block sau cấp phát
  - `Fragment = block.size - file.size`
  - Không thể sử dụng được nếu file sau lớn hơn fragment
- **Phân mảnh trong (Internal Fragmentation)**: Khoảng trống trong block được cấp phát
  - Xảy ra khi cấp phát cố định (không phải variable allocation)

### Locality of Reference
- **Khái niệm**: CPU thường truy cập bộ nhớ gần với vị trí truy cập trước đó
- **Ý nghĩa**: Giải thích tại sao LRU hoạt động tốt
- **Ứng dụng**: Cơ sở để thiết kế cache, virtual memory

### Belady's Anomaly
- **Khái niệm**: Tăng số frame (bộ nhớ) nhưng lại tăng page fault
- **Xảy ra ở**: FIFO
- **Không xảy ra ở**: LRU, LFU, Optimal
- **Ví dụ**: Với FIFO, frame=3 có 11 page fault, nhưng frame=4 có 12 page fault

## IV. CÂU HỎI ÔN TẬP

### Cấp Phát Bộ Nhớ
1. Thuật toán nào cho fragment nhỏ nhất? **Best Fit**
2. Thuật toán nào nhanh nhất? **First Fit**
3. Khi nào dùng Worst Fit? **Hiếm khi, hầu như không bao giờ**
4. Fragment = ? **block.size - file.size**

### Thay Thế Trang
1. Thuật toán nào cho page fault ít nhất trên lý thuyết? **Optimal**
2. Thuật toán nào thực tiễn tốt nhất? **LRU**
3. FIFO bị anomaly nào? **Belady's Anomaly**
4. MFU dùng khi nào? **Hiếm khi, không thực tế**
5. Sự khác nhau LRU vs LFU?
   - **LRU**: Xét **lần truy cập cuối cùng** (thời gian)
   - **LFU**: Xét **số lần truy cập** (tần suất)

## V. KẾT LUẬN

### Khuyến Cáo Sử Dụng

**Cấp Phát Bộ Nhớ:**
```
Nếu cần tốc độ cao       → First Fit
Nếu cần tiết kiệm bộ nhớ  → Best Fit ✓ (Khuyến cáo)
Nếu cần thử nghiệm        → Worst Fit (không dùng thực tế)
```

**Thay Thế Trang:**
```
Nếu cần lý thuyết tối ưu  → Optimal (chỉ lý thuyết)
Nếu cần thực tiễn tốt      → LRU ✓ (Khuyến cáo)
Nếu cần nhanh & đơn giản   → FIFO (nhưng tệ hơn LRU)
Nếu hành động không đều    → LFU (nhưng phức tạp)
Tránh sử dụng              → MFU (không thực tế)
```

    vector<int> fragment(np, 0);         // Lưu fragment

            fragment[i] = minFragment;
            used[bestIdx] = true;
        }

### 4. FIFO - FIFO.cpp

#### 📝 Giải Thích Code
```cpp
#define FRAMES_NUMBER 3     // Số frame = 3

int FIFO(int pages[], int size) {
    int frames[FRAMES_NUMBER];
    for (int i = 0; i < FRAMES_NUMBER; i++)
        frames[i] = -1;     // Khởi tạo: -1 = frame rỗi

    int paeFaults = 0;      // Đếm page fault
    int victim = 0;         // Index của frame sẽ bị thay thế (FIFO queue)

    for (int pageIndex = 0; pageIndex < size; pageIndex++) {
        // Bước 1: Tìm trang trong frames
        bool isFound = false;
        for (int i = 0; i < FRAMES_NUMBER; i++)
            if (frames[i] == pages[pageIndex]) {
                isFound = true;
                cout << pages[pageIndex] << endl;  // Trang đã có → không lỗi
                break;
            }

        // Bước 2: Nếu không tìm thấy
        if (!isFound) {
            bool hasFreeFrame = false;
            
            // Tìm frame rỗi
            for (int i = 0; i < FRAMES_NUMBER; i++)
                if (frames[i] == -1) {
                    hasFreeFrame = true;
                    frames[i] = pages[pageIndex];
                    paeFaults++;
                    
                    cout << pages[pageIndex] << "\t\t";
                    for (int f = 0; f < FRAMES_NUMBER; f++)
                        cout << frames[f] << "\t";
                    cout << endl;
                    break;
                }

            // Bước 3: Nếu không có frame rỗi → thay thế (FIFO)
            if (!hasFreeFrame) {
                frames[victim] = pages[pageIndex];  // Thay thế tại vị trí victim
                victim = (victim + 1) % FRAMES_NUMBER;  // Chuyển victim sang frame tiếp theo
                paeFaults++;

                cout << pages[pageIndex] << "\t\t";
                for (int f = 0; f < FRAMES_NUMBER; f++)
                    cout << frames[f] << "\t";
                cout << endl;
            }
        }
    }
    return paeFaults;
}
```

**Các biến quan trọng:**
- `frames[3]`: 3 frame, mỗi frame chứa một trang (hoặc -1 nếu rỗi)
- `victim`: Index của frame sẽ bị thay thế tiếp theo (xoay vòng)
- `paeFaults`: Số lần page fault

### 5. LRU - LRU.cpp

#### 📝 Giải Thích Code
```cpp
// Khác biệt: Tìm lần truy cập cuối cùng (lastUse)

if (!hasFreeFrame) {
    int lastUse[FRAMES_NUMBER];
    
    // Tìm lần truy cập cuối cùng của mỗi trang
    for (int i = 0; i < FRAMES_NUMBER; i++)
        for (int p = pageIndex; p >= 0; p--)  // ← Tìm từ hiện tại ngược lại
            if (pages[p] == frames[i]) {
                lastUse[i] = p;        // Lưu index lần truy cập gần nhất
                break;
            }

    // Tìm frame có lastUse nhỏ nhất (lâu nhất)
    int victim = 0;
    for (int i = 0; i < FRAMES_NUMBER; i++)
        if (lastUse[i] < lastUse[victim])  // ← Index nhỏ = xa hơn trong quá khứ
            victim = i;

    frames[victim] = pages[pageIndex];
    paeFaults++;
}
```

**Logic khác LRU:**
- Tìm từng trang trong frames
- Duyệt ngược từ `pageIndex` đến 0 để tìm lần truy cập gần nhất
- Lưu index của lần truy cập gần nhất
- Chọn trang có index **nhỏ nhất** = truy cập lâu nhất trong quá khứ

// Khác biệt: Đếm số lần truy cập (countUse)

if (!hasFreeFrame) {

### 8. OPTIMAL - Optimal.cpp

#### 📝 Giải Thích Code
```cpp
// Khác biệt: Tìm lần truy cập TIẾP THEO xa nhất

if (!hasFreeFrame) {
    int nextUse[FRAMES_NUMBER];
    for (int i = 0; i < FRAMES_NUMBER; i++)
        nextUse[i] = size + 1;  // ← Mặc định: không bao giờ sử dụng

    // Tìm lần truy cập tiếp theo của mỗi trang
    for (int i = 0; i < FRAMES_NUMBER; i++)
        for (int p = pageIndex; p < size; p++)  // ← Tìm từ hiện tại trở đi (tương lai)
            if (pages[p] == frames[i]) {
                nextUse[i] = p;  // Lưu index lần truy cập tiếp theo (sớm nhất)
                break;
            }

    // Tìm frame có nextUse lớn nhất (sử dụng muộn nhất / không bao giờ)
    int victim = 0;
    for (int i = 0; i < FRAMES_NUMBER; i++)
        if (nextUse[i] > nextUse[victim])  // ← Index lớn = xa hơn trong tương lai
            victim = i;

    frames[victim] = pages[pageIndex];
    paeFaults++;
}
```

**Logic khác Optimal:**
- Duyệt **tương lai** (từ `pageIndex` đến `size`)
- Tìm lần truy cập tiếp theo **sớm nhất** của mỗi trang
- Chọn trang có nextUse **lớn nhất** = sẽ được sử dụng muộn nhất

### 6. LFU - LFU.cpp

#### 📝 Giải Thích Code
```cpp
// Khác biệt: Đếm số lần truy cập (countUse)

if (!hasFreeFrame) {
    int countUse[FRAMES_NUMBER] = {0};
    
    // Đếm số lần mỗi trang trong frames được truy cập
    for (int i = 0; i < FRAMES_NUMBER; i++)
        for (int p = pageIndex; p >= 0; p--)  // ← Duyệt tất cả trước đó
            if (pages[p] == frames[i])
                countUse[i]++;  // ← Tăng đếm mỗi khi tìm thấy

    // Tìm frame có countUse nhỏ nhất (ít dùng nhất)
    int victim = 0;
    for (int i = 0; i < FRAMES_NUMBER; i++)
        if (countUse[i] < countUse[victim])  // ← Số lần dùng ít nhất
            victim = i;

    frames[victim] = pages[pageIndex];
    paeFaults++;
}
```

**Logic khác LFU:**
- Đếm **tất cả** lần xuất hiện của trang trong quá khứ (không dừng ở lần đầu tiên)
- Chọn trang có **số lần dùng ít nhất** để thay thế

#### 📤 Output Ví Dụ
```
Sequence: 1, 2, 3, 1, 4, 5, 2, 1, 2, 6, 7, 3, 2

Bước 1-3: [1, 2, 3] countUse = [1, 1, 1] → 3 faults
Bước 4: [1, 2, 3] countUse = [2, 1, 1] → 0 faults (1 đã có)
Bước 5: Trang 2 hoặc 3 (countUse=1) thay thế → Chọn 2 → [1, 4, 3] countUse = [2, 1, 1]
...
Total Page Faults = 9
```
