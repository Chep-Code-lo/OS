## 1️. FCFS (First Come First Served)

### Ý tưởng:
Process nào đến trước thì được thực thi trước, giống như xếp hàng.

### Mã giả:

```
Bước 1: Chuẩn bị
    - Sắp xếp tất cả processes theo arrival_time (tăng dần)
    - current_time = 0
    - completed = 0

Bước 2: Lặp qua từng process theo thứ tự đã sắp xếp
    Với mỗi process p:
        
        // Nếu CPU đang rảnh (current_time < arrival_time)
        Nếu current_time < p.arrival_time:
            current_time = p.arrival_time
        
        // Tính thời gian chờ
        p.waiting_time = current_time - p.arrival_time
        
        // Process bắt đầu chạy
        p.response_time = current_time - p.arrival_time
        
        // Thực thi process (cộng burst_time)
        current_time = current_time + p.burst_time
        
        // Process kết thúc
        p.return_time = current_time
        p.turnaround_time = p.return_time - p.arrival_time
        
        completed = completed + 1

Bước 3: Kết thúc
    - In kết quả
    - Tính average waiting time, turnaround time
```

### Ví dụ minh họa:

```
Input:
P1: arrival=0, burst=5
P2: arrival=1, burst=3
P3: arrival=2, burst=2

Thực thi:
t=0: P1 bắt đầu (waiting=0)
t=5: P1 xong, P2 bắt đầu (waiting=4)
t=8: P2 xong, P3 bắt đầu (waiting=6)
t=10: P3 xong

Kết quả:
P1: waiting=0, turnaround=5
P2: waiting=4, turnaround=7
P3: waiting=6, turnaround=8
```

## 2️⃣ SJF - Non-preemptive (Shortest Job First)

### Ý tưởng:
Chọn process có burst_time nhỏ nhất trong các process đã đến.

### Mã giả:

```
Bước 1: Chuẩn bị
    - Sắp xếp processes theo arrival_time
    - current_time = 0
    - completed = 0
    - Tạo mảng is_completed[] = false cho tất cả process

Bước 2: Lặp cho đến khi tất cả process hoàn thành
    Lặp (completed < tổng số process):
        
        // Tìm process có burst_time nhỏ nhất
        min_burst = VÔ CỰC
        selected_process = NULL
        
        Với mỗi process p:
            Nếu (p chưa hoàn thành) VÀ 
               (p.arrival_time <= current_time) VÀ 
               (p.burst_time < min_burst):
                min_burst = p.burst_time
                selected_process = p
        
        // Nếu tìm thấy process
        Nếu selected_process != NULL:
            p = selected_process
            
            // Tính thời gian chờ
            p.waiting_time = current_time - p.arrival_time
            
            // Tính response time (lần đầu được CPU)
            p.response_time = current_time - p.arrival_time
            
            // Thực thi process
            current_time = current_time + p.burst_time
            
            // Process kết thúc
            p.return_time = current_time
            p.turnaround_time = p.return_time - p.arrival_time
            p.is_completed = true
            completed = completed + 1
        
        // Nếu không tìm thấy (CPU idle)
        Ngược lại:
            current_time = current_time + 1

Bước 3: Kết thúc
```

### Ví dụ minh họa:

```
Input:
P1: arrival=0, burst=7
P2: arrival=2, burst=4
P3: arrival=4, burst=1
P4: arrival=5, burst=4

Thực thi:
t=0: Chọn P1 (burst=7, duy nhất đã đến)
t=7: P1 xong
     Đã đến: P2(4), P3(1), P4(4)
     Chọn P3 (burst nhỏ nhất)
t=8: P3 xong
     Đã đến: P2(4), P4(4)
     Chọn P2 hoặc P4 (bằng nhau, chọn đến trước)
t=12: P2 xong
t=16: P4 xong
```

## 3️⃣ SJF - Preemptive (Shortest Remaining Time)

### Ý tưởng:
Mỗi đơn vị thời gian, chọn process có remaining_time nhỏ nhất. Process mới đến có thể ngắt process đang chạy.

### Mã giả:

```
Bước 1: Chuẩn bị
    - Sắp xếp processes theo arrival_time
    - current_time = 0
    - completed = 0
    - Tạo mảng remain_time[] = burst_time[] cho tất cả process
    - Tạo mảng first_run[] = false cho tất cả process

Bước 2: Lặp cho đến khi tất cả process hoàn thành
    Lặp (completed < tổng số process):
        
        // Tìm process có remain_time nhỏ nhất
        min_remain = VÔ CỰC
        selected_process = NULL
        
        Với mỗi process p:
            Nếu (p.arrival_time <= current_time) VÀ 
               (remain_time[p] > 0) VÀ 
               (remain_time[p] < min_remain):
                min_remain = remain_time[p]
                selected_process = p
        
        // Nếu tìm thấy process
        Nếu selected_process != NULL:
            p = selected_process
            
            // Lần đầu chạy → tính response time
            Nếu first_run[p] == false:
                p.response_time = current_time - p.arrival_time
                first_run[p] = true
            
            // Thực thi 1 đơn vị thời gian
            remain_time[p] = remain_time[p] - 1
            current_time = current_time + 1
            
            // Nếu process hoàn thành
            Nếu remain_time[p] == 0:
                p.return_time = current_time
                p.turnaround_time = current_time - p.arrival_time
                p.waiting_time = p.turnaround_time - p.burst_time
                completed = completed + 1
        
        // Nếu không tìm thấy (CPU idle)
        Ngược lại:
            current_time = current_time + 1

Bước 3: Kết thúc
```

### Ví dụ minh họa:

```
Input:
P1: arrival=0, burst=8
P2: arrival=1, burst=4
P3: arrival=2, burst=2

Thực thi chi tiết:
t=0: P1 chạy (remain=8)
t=1: P2 đến (remain=4), P1(remain=7)
     Chọn P2 (4 < 7) → ngắt P1
t=5: P2 xong, P1(remain=7), P3(remain=2)
     Chọn P3 (2 < 7)
t=7: P3 xong, P1(remain=7)
     Chọn P1
t=14: P1 xong

Timeline:
0-1: P1
1-5: P2 (ngắt P1)
5-7: P3
7-14: P1 (tiếp tục)
```

## 4️⃣ Round Robin

### Ý tưởng:
Mỗi process được cấp quantum đơn vị thời gian. Nếu chưa xong → đưa vào cuối hàng đợi.

### Mã giả:

```
Bước 1: Chuẩn bị
    - Sắp xếp processes theo arrival_time
    - current_time = 0
    - completed = 0
    - quantum = giá trị quantum (ví dụ: 2)
    - Tạo mảng remain_time[] = burst_time[]
    - Tạo mảng first_run[] = false
    - Tạo Queue (hàng đợi)

Bước 2: Thêm process đầu tiên vào Queue
    Thêm process đầu tiên vào Queue

Bước 3: Lặp cho đến khi Queue rỗng
    Lặp (Queue không rỗng):
        
        // Lấy process từ đầu Queue
        p = Queue.dequeue()
        
        // Lần đầu chạy → tính response time
        Nếu first_run[p] == false:
            p.response_time = current_time - p.arrival_time
            first_run[p] = true
        
        // Tính thời gian thực thi
        exec_time = min(quantum, remain_time[p])
        
        // Thực thi
        current_time = current_time + exec_time
        remain_time[p] = remain_time[p] - exec_time
        
        // Thêm các process mới đến vào Queue
        Với mỗi process q:
            Nếu (q.arrival_time <= current_time) VÀ 
               (q chưa trong Queue) VÀ 
               (remain_time[q] > 0):
                Queue.enqueue(q)
        
        // Nếu process chưa xong → đưa vào cuối Queue
        Nếu remain_time[p] > 0:
            Queue.enqueue(p)
        
        // Nếu process xong
        Ngược lại:
            p.return_time = current_time
            p.turnaround_time = current_time - p.arrival_time
            p.waiting_time = p.turnaround_time - p.burst_time
            completed = completed + 1

Bước 4: Kết thúc
```

### Ví dụ minh họa:

```
Input (Quantum = 2):
P1: arrival=0, burst=5
P2: arrival=1, burst=3
P3: arrival=2, burst=1

Thực thi:
t=0: Queue=[P1]
     P1 chạy 2s (remain=3)
     Queue=[P2, P1]

t=2: Queue=[P2, P1]
     P2 chạy 2s (remain=1)
     Queue=[P1, P3, P2]

t=4: Queue=[P1, P3, P2]
     P1 chạy 2s (remain=1)
     Queue=[P3, P2, P1]

t=6: Queue=[P3, P2, P1]
     P3 chạy 1s (xong)
     Queue=[P2, P1]

t=7: Queue=[P2, P1]
     P2 chạy 1s (xong)
     Queue=[P1]

t=8: Queue=[P1]
     P1 chạy 1s (xong)
     Queue=[]
```

## 5️⃣ Priority - Non-preemptive

### Ý tưởng:
Chọn process có priority cao nhất (số nhỏ nhất). Không ngắt process đang chạy.

### Mã giả:

```
Bước 1: Chuẩn bị
    - Sắp xếp processes theo arrival_time
    - current_time = 0
    - completed = 0
    - Tạo mảng is_completed[] = false

Bước 2: Lặp cho đến khi tất cả process hoàn thành
    Lặp (completed < tổng số process):
        
        // Tìm process có priority cao nhất (số nhỏ nhất)
        min_priority = VÔ CỰC
        selected_process = NULL
        
        Với mỗi process p:
            Nếu (p chưa hoàn thành) VÀ 
               (p.arrival_time <= current_time) VÀ 
               (p.priority < min_priority):
                min_priority = p.priority
                selected_process = p
            
            // Nếu priority bằng nhau → chọn đến trước
            Nếu (p.priority == min_priority) VÀ 
               (p.arrival_time < selected_process.arrival_time):
                selected_process = p
        
        // Nếu tìm thấy process
        Nếu selected_process != NULL:
            p = selected_process
            
            // Tính thời gian
            p.waiting_time = current_time - p.arrival_time
            p.response_time = current_time - p.arrival_time
            
            // Thực thi
            current_time = current_time + p.burst_time
            
            // Kết thúc
            p.return_time = current_time
            p.turnaround_time = p.return_time - p.arrival_time
            p.is_completed = true
            completed = completed + 1
        
        // CPU idle
        Ngược lại:
            current_time = current_time + 1

Bước 3: Kết thúc
```

### Ví dụ minh họa:

```
Input (số nhỏ = priority cao):
P1: arrival=0, burst=4, priority=3
P2: arrival=1, burst=3, priority=1
P3: arrival=2, burst=2, priority=2

Thực thi:
t=0: Chọn P1 (duy nhất đã đến)
t=4: P1 xong
     Đã đến: P2(priority=1), P3(priority=2)
     Chọn P2 (priority cao nhất)
t=7: P2 xong
     Chọn P3
t=9: P3 xong

Thứ tự: P1 → P2 → P3
```

## 8️⃣ Multilevel Feedback Queue (MLFQ)

### Ý tưởng:
3 queue với quantum khác nhau. Process mới vào Q0, dùng hết quantum → xuống queue thấp hơn.

### Mã giả:

```
Bước 1: Chuẩn bị
    - current_time = 0
    - completed = 0
    - Tạo mảng remain_time[] = burst_time[]
    - Tạo mảng queue_level[] = 0 (tất cả bắt đầu ở Q0)
    - Tạo mảng first_run[] = false
    - quantum[0] = 2
    - quantum[1] = 4
    - quantum[2] = VÔ CỰC (FCFS)

Bước 2: Lặp cho đến khi tất cả process hoàn thành
    Lặp (completed < tổng số process):
        
        // Tìm process theo thứ tự ưu tiên queue
        selected_process = NULL
        
        // Tìm trong Q0 trước
        Với mỗi process p có queue_level[p] == 0:
            Nếu (p.arrival_time <= current_time) VÀ (remain_time[p] > 0):
                selected_process = p
                break
        
        // Nếu Q0 rỗng → tìm trong Q1
        Nếu selected_process == NULL:
            Với mỗi process p có queue_level[p] == 1:
                Nếu (p.arrival_time <= current_time) VÀ (remain_time[p] > 0):
                    selected_process = p
                    break
        
        // Nếu Q1 rỗng → tìm trong Q2
        Nếu selected_process == NULL:
            Với mỗi process p có queue_level[p] == 2:
                Nếu (p.arrival_time <= current_time) VÀ (remain_time[p] > 0):
                    selected_process = p
                    break
        
        // Nếu tìm thấy process
        Nếu selected_process != NULL:
            p = selected_process
            level = queue_level[p]
            
            // Lần đầu chạy
            Nếu first_run[p] == false:
                p.response_time = current_time - p.arrival_time
                first_run[p] = true
            
            // Tính thời gian thực thi
            Nếu level == 2:
                // Q2: FCFS (chạy hết)
                exec_time = remain_time[p]
            Ngược lại:
                // Q0, Q1: dùng quantum
                exec_time = min(quantum[level], remain_time[p])
            
            // Thực thi
            current_time = current_time + exec_time
            remain_time[p] = remain_time[p] - exec_time
            
            // Nếu hoàn thành
            Nếu remain_time[p] == 0:
                p.return_time = current_time
                p.turnaround_time = current_time - p.arrival_time
                p.waiting_time = p.turnaround_time - p.burst_time
                completed = completed + 1
            
            // Nếu dùng hết quantum → xuống queue thấp hơn
            Ngược lại nếu (level < 2) VÀ (exec_time == quantum[level]):
                queue_level[p] = level + 1
        
        // CPU idle
        Ngược lại:
            current_time = current_time + 1

Bước 3: Kết thúc
```

### Ví dụ minh họa:

```
Input:
P1: arrival=0, burst=10

Thực thi:
t=0: P1 ở Q0 (quantum=2)
     P1 chạy 2s (remain=8)
     Dùng hết quantum → xuống Q1

t=2: P1 ở Q1 (quantum=4)
     P1 chạy 4s (remain=4)
     Dùng hết quantum → xuống Q2

t=6: P1 ở Q2 (FCFS)
     P1 chạy 4s (xong)

Timeline:
Q0: 0-2 (2s)
Q1: 2-6 (4s)
Q2: 6-10 (4s)
```

## 📐 Công thức tính toán

```
waiting_time = return_time - arrival_time - burst_time

turnaround_time = return_time - arrival_time

response_time = first_execution_time - arrival_time

return_time = thời điểm process kết thúc
```

## 🔑 Điểm quan trọng

### 1. Xử lý CPU Idle
```
Nếu không có process nào đã đến hoặc tất cả đã hoàn thành:
    current_time = current_time + 1
```

### 2. Kiểm tra Process đã đến
```
Nếu p.arrival_time <= current_time:
    // Process đã đến, có thể xử lý
```

### 3. Phân biệt Preemptive vs Non-preemptive

**Non-preemptive:**
- Chạy đến hết: `current_time += burst_time`
- Không cần `remain_time[]`

**Preemptive:**
- Chạy từng đơn vị: `current_time += 1`
- Cần `remain_time[]`

### 4. Response Time
```
// Chỉ tính lần đầu tiên process được CPU
Nếu first_run[p] == false:
    p.response_time = current_time - p.arrival_time
    first_run[p] = true
```
